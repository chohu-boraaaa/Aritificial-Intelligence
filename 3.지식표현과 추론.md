# 지식표현
- 인간의 지식을 이용한 인공지능 구현을 위해 필요
- 프로그램을 통한 구현을 위해 정형적 표현과 처리(추론) 방법 필요

## 데이터 피라미드
- 데이터
- 정보
- 지식
- 지혜

## 지식
- 암묵지 : 형식을 갖춰 표현하기 어려운, 학습과 경험을 통해 쌓은 지식
- 형식지 : 비교적 쉽게 형식을 갖춰 표현될 수 있는 지식
- 절차적 지식 : 문제해결의 절차 기술
- 선언적 지식 : 어떤 대상의 성질, 특성, 관계 기술

## 인공지능에서의 지식 표현 및 처리
- 정형화된 형태로 표현

## 규칙
- 조건부의 지식 표현, IF-THEN 형태
- 직관적, 이해하기 쉬움
- IF : 주어진 정보나 사실에 대응된 조건 (조건부)
- THEN : 조건부가 만족될 때 판단이나 행동 (결론부)

### 규칙 획득 및 표현
- 대상
- 속성
- 행동/판단

### 규칙의 구성
- 조건부 : 둘 이상 조건을 AND나 OR로 연결 가능
- 결론부 : 여러 개의 판단 또는 행동 포함

### 규칙을 통한 지식 표현
- 인과관계 : 원인을 조건부에 결과를 결론부에 표현
- 추천 : 상황을 조건부에 추천내용을 결론부에
- 지시 : 상황을 조건부에 지시내용을 결론부에
- 전략 : 일련의 규칙들로 표현, 이전 단계의 판정결과에 의해 다음 단계에 고려할 규칙이 결정
- 휴리스틱 : 경험적 지식 표현, 전문적인 지식은 항상 최적을 보장하는 것이 아닌 일반적으로 바람직한 것을 표현

## 프레임
- 특정 객체 또는 개념에 대한 전형적인 지식을 슬롯의 집합으로 표현하는 것 (프레임 : 트리, 슬롯 : 끼워넣는 것)

### 슬롯
- 객체의 속성을 기술하는 것
- 슬롯 이름과 슬롯 값으로 구성
- 슬롯 값 : 복수의 패싯(facet)과 데몬(demon)으로 구성

### 패싯(facet)
- 속성에 대한 부가적 정보 기술
- 패시이름과 패싯 값의 쌍으로 구성
- value : 속성값 (수, 문자열, 다른 프레임의 포인터)
- data-type : 속성값의 자료형
- default : 디폴트값(속성값이 주어지지 않을 때 사용되는 초기값)
- require : 슬롯에 들어갈 수 있는 값이 만족해야 할 제약조건

### 데몬(demon)
- 지정된 조건을 만족할 때 실행할 절차적 지식
- 슬롯 값으로 데몬 실행조건과 데몬 이름의 쌍

#### 데몬 실행조건의 예
- if_needed : 슬롯 값을 알아야 할 때(사용하려고 할 때)
- if_added : 슬롯 값이 추가될 때
- if_removed : 슬롯 값이 제거될 때
- if_modified : 슬롯 값이 수정될 때

### 프레임의 종류
- 클래스 프레임 : 부류(class)에 대한 정보 표현
- 인스턴스 프레임 : 객체에 대한 정보 표현 (인스턴스는 클래스 프레임을 상속받을 수 있음)

### 프레임 계층 구조
- 상위 프레임 : 클래스를 나타내는 프레임
- 하위 프레임 : 상위 프레임의 객체, 상위 프레임을 상속받음

### 프레임과 규칙을 결합한 지식표현
- 규칙을 사용하여 조건적인 지식 표현 : 데몬에 규칙 사용, 규칙의 조건부나 결론부에서 프레임 사용
- 대부분의 규칙기반 시스템에서 객체 개념 사용 : 객체의 표현에 프레임 사용 가능

## 규칙 기반 시스템
- 지식을 규칙 형태로 표현
- 주어진 문제 상황에 적용될 수 있는 규칙들을 사용하여 문제에 대한 해를 찾도록 하는 지식 기반 시스템(KBS)
- 전문가 시스템을 구현하는 전형적 시스템 : 특정 문제영역에서 전문가 수준의 해를 찾아주는 시스템

### 추론
- 추론 : 구축된 지식과 정보로 새로운 사실을 생성하는 것

#### 전향추론(순방향추론)
- 규칙의 조건부와 조건을 만족하는 사실이 있을 때 결론부 실행하거나 처리

#### 후향추론(역방향추론)
- 어떤 사실을 검증하거나 확인하고 싶을 때 결론부에 가지고 있는 규칙들을 찾아서 조건부의 조건들이 모두 만족하는지 확인

### 규칙 기반 시스템 구조
- 규칙 : 문제 해결을 위한 지식
- 사실 : 문제 영역에 대해 알려진 정보나 데이터

#### 규칙 베이스
- 전체 규칙의 집합 관리. 생성 메모리(production memory) 라고도 함.

#### 작업 메모리
- 사용자로부터 받은 정보 관리, 중간 결과 저장, 유도된 최종해 저장. 작업 메모리에 저장되는 모든 것 사실.

#### 추론 엔진
- 실행할 수 있는 규칙을 찾아서 해당 규칙을 실행하는 역할, 패턴 매칭 - 경합 해소 - 규칙 실행 반복
- 패턴 매칭 : 작업 메모리의 사실과 규칙 베이스의 규칙의 조건부를 대조하여 일치하는 규칙을 찾는 과정
- 경합 집합 : 실행가능한 규칙들의 집합
- 경합 해소 : 경합집합에서 하나의 규칙 선택
- 사용자 인터페이스 : 규칙베이스 및 작업 메모리 관리, 추론 엔진 조작
- 외부 인터페이스 : 외부 데이터나 함수의 기능 지원

##### 경합해소 전략
- 규칙 우선 순위 : 미리 각 규칙에 우선순위 부여하여 가장 높은 규칙 선택
- 최신 우선 : 가장 최근에 입력된 데이터와 매칭된 규칙 선택
- 최초 우선 : 경합 집합에서 가장 먼저 매칭된 규칙 선택
- 상세 우선 : 가장 상세한 조건부를 갖는 규칙 선택

## 의미망
- 지식을 이항 관계 집합으로 표현, 노드와 간선을 이용하여 지식 표현

### 노드
- 대상, 행위, 개념, 상태, 사건

### 간선
- 관계가 있는 노드 연결
- 관계에 따른 방향
- 관계의 의미를 나타내는 라벨 부여

### 의미망에서 사용되는 관계
- is-a : 상위 클래스와 하위 클래스의 관계, 클래스와 객체의 관계를 나타내어 계층 관계 표현, 상위 계층 상속, 추이적 관계 만족
- has-a : 전체-부분 관계, part-of 역관계, 추이적 관계 만족

### 다항관계를 이항관계로 전개하여 표현한 의미망
- 사물화 : 관계를 객체로 간주하여 노드로 표현

### 의미망에서의 추론
- 상속 이용
- 질문에 대한 의미망과 지식을 나타내는 의미망 비교
- 디폴트 값 이용한 추론
- 주어진 사실로부터 새로운 사실 이끌어내는 추론

### 의미망 표현의 장점
- 시각적으로 표현하여 직관적 이해 가능
- 노드 추가 및 변경으로 쉽게 지식의 추가 및 변경 가능
- 개념의 계층관계 정의하여 상속 관계 지정 가능
- 복잡한 지식 구조화하여 표현

### 의미망 표현의 단점
- 지식의 양이 많아지면 복잡
- 개념이나 관계를 임의로 정의하기 때문에 통일성 부족
- 공유나 재사용에 대한 고려 X
- 논리적 결합관계나 인과 관계 표현하려면 링크 도입을 해야 해서 추론과정을 복잡하게 만듦
- 정적인 지식 표현 : 지식의 내용을 바꾸려면 그래프를 동적으로 바꿀 수 있도록 해야 함

## 스크립트
- 전형적인 상황에 대한 절차적 지식으로 표현, 일련의 사건들을 시간적 순서를 고려하여 기술

### 구성요소
- 1. 진입조건 : 스크립트에 기술된 사건들이 일어나기 전에 만족되어야 하는 조건
- 2. 역할자 : 스크립트에 관련된 사람이나 대상
- 3. 자산 : 사건 진행과정에서 사용되는 객체
- 4. 트랙 : 어떤 스크립트에서 발생할 수 있는 일련의 사건들의 변형된 형태 식별자
- 5. 장면 : 실제 일어나는 일련의 사건
- 6. 결과 조건 : 스크립트에 있는 장면들이 일어난 이후에 만족되는 조건

## 온톨로지
- 영역에 대한 공통된 어휘 및 이해 지원
- 어떤 영역에 대한 지식을 공유 및 재사용할 수 있도록 해당영역의 개념과 관계를 나타내는 어휘를 정의하고 이를 사용해 지식을 표현해 놓은 것
- 서로 토의 해 합의에 이른 것
- 컴퓨터에서 다룰 수 있도록 정형화해서 표현
- 사례 : dmoz(분야별로 정리한 디렉토리 서비스), UNSPSC(제품 및 서비스 용어), WordNet(영어 단어의 어휘목록, 어휘목록 사이의 관계), UMLS(의료 영역 용어 체계), SNOMED-CT(의료용어, 유사어, 코드), LOINC(병원 검사 명칭 및 임상 용어 표준 체계), GO(유전자와 유전자 산물0

### 온톨로지의 지식 표현
- 의미망으로 표현 가능
- RDF : 자원에 대한 메타데이터를 기술하는 명세, 특정 대상을 기술하는 간단한 문장, 주어+서술어+목적어, 자원(대상)-속성(관계)-값으로 분해
- RDFS : RDF를 사용하여 온톨로지를 표현할 때 관련 어휘 온톨로지를 정의하는 언어
- SARQL : RDF 형태로 저장된 질의어, SQL과 유사한 문법
- RIF : 규칙을 정의하고 교환하기 위한 규약
- OWL : 웹 상의 자원과 이들의 속성에 대한 지식을 표현하기 위한 온톨로지 언어

### 온톨로지의 활용 : 시멘틱 웹
- 웹의 데이터를 소프트웨어 에이전트가 이해하여 지능적으로 활용할 수 있도록 하는 것
- 처음 설계 : Tim Bernes-Lee
- 의미있는 태그를 정의하여 문서를 기술하기 위해 XML 활용
- 태그 및 데이터의 의미해석 위해 RDF이용
- 온톨로지 구축을 통해 태그 및 메타데이터 의미 해석

### 의미망과 온톨로지

#### 의미망
- 용어가 임의적
- 통일된 표현 관련 규정 부재

#### 온톨로지
- 공유 및 상호운영성
- 정보 및 지식의 재사용성

### 온톨로지 구축 도구

## 지식 그래프

- 노드 : 개체
- 간선(edge) : 관계

## 논리
- 말로 표현된 문장들에 기호를 부여하고 해당 기호의 조작으로 참 거짓을 판단하는 분야

## 명제 논리
- 명제 : 참, 거짓 따질 수 있는 문장
- 명제 기호의 진리값을 사용하여 명제들에 의해 표현되는 문자들의 진리값 결정
- 문장 자체의 내용에 대해서는 무관심, 문장의 진리값에만 관심

### 기본 명제
- 하나의 진술로 이루어진 최소 단위의 명제

### 복합 명제
- 기본 명제들이 결합되어 만들어진 명제

### 명제 논리의 구문
- 논리식 : 명제를 기호로 표현한 형식
- 리터럴 : 명제 기호와 명제 기호의 부정
- 절 : 리터럴들이 논리합으로만 연결되거나 논리곱으로만 연결되는 논리식
- 논리곱 정규형(CNF) : 논리합 절들이 논리곱으로 연결되어 있는 논리식
- 논리합 정규형(DNF) : 논리곱 절들이 논리합으로 연결되어 있는 논리식
- 정형식(wff) : 문법에 맞는 논리식

### 명제 논리의 의미
- 진리표 : 논리 기호에 따라 참과 거짓을 결합하는 방법을 나타낸 표
- 명제기호의 외연/의미 : 명제 기호에 명제를 대응 시킨 것
- 논리식의 모델 : 각 명제 기호에 T,F로 표현될 수 할당한 것, n개의 명제기호가 논리식에 사용된다면 각각 T,F의 값을 가질 수 있기 때문에 2^n개의 모델 존재
- 타당한 논리식(=항진식) : 모든 가능한 모델에 대해 항상 참인 논리식
- 항위식 : 모든 가능한 모델에 대해 항상 거짓인 논리식
- 충족가능한 논리식 : 참으로 만들 수 있는 논리식이 하나라도 있는 논리식
- 충족불가능한 논리식(=항위식) : 참으로 만들 수 있는 논리식이 전혀 없는 논리식
- 동치 관계의 논리식 : 좌변과 우변이 항상 같은 논리식
- 논리적 귀결 : 정형식의 집합에 있는 모든 정형식을 참으로 만든다면 정형식을 참으로 만든다. = 정형식의 집합은 정형식을 논리적으로 귀결 = 정형식은 정형식의 집합을 논리적으로 따른다 = 정형식은 정형식의 집합의 논리적 결론이다. 

### 명제 논리의 추론
- 귀납적 추론 : 복수의 사실들을 일반화
- 연역적 추론 : 참인 사실들 또는 명제로부터 새로운 참인 사실 또는 명제를 도출
- 논리에서의 추론 : 함의의 논리적 관계를 이용하여 새로운 논리식 유도

#### 추론규칙
- 참인 논리식들이 논리적으로 귀결하는 새로운 논리식을 만들어내는 기계적으로 적용되는 규칙
- 긍정논법 : 새는 날 수 있다, 새 |- 날 수 있다.
- 부정논법 : 새는 날 수 있다, 날수 없다 |- 새가 아니다.
- 삼단논법 : 새는 날개가 있다. 날개가 있으면 날 수 있다, 새이면 |- 날 수 있다.
- 논리융합 : 두개의 논리합 절에서 같은 명제기호의 긍정과 부정의 리터럴을 포함하고 있을 때 해당 리터럴 들을 제외한 나머지의 논리합절 생성

#### 추론규칙의 정당성과 완전성
- 정당성 : 추론규칙에 의해 생성된 논리식은 항상 논리적으로 귀결한다.
- 완전성 : 논리적으로 귀결하는 논리식은 항상 추론규칙을 찾을 수 있다.

#### 정리증명
- 공리 : 추론을 할 때 참으로 주어진 논리식
- 정리 : 공리들에 추론규칙을 적용하여 얻어지는 논리식

##### 정리 증명
- 구성적 증명 : 공리들에 추론규칙을 적용하여 증명해 보이는 것
- 논리융합 반박 : 정리를 부정한 다음 논리융합을 이용하여 모순이 발생하는 것을 보여 정리가 참임을 증명

### 명제논리의 지식표현

#### 명제논리를 이용한 지식표현 절차
- 문장으로 표현된 지식으로부터 기본 명제들 추출
- 명제들을 명제기호로 할당
- 논리적 연결 관계를 참고하여 명제기호들을 논리기호로 연결하여 논리식 구성

#### 명제논리로 표현된 지식에 대한 추론
- 명제 기호가 나타내는 명제의 의미와는 무관
- 대수적인 기호연산을 통해 추론 수행

## 술어 논리 기반 지식표현

### 술어 논리
- 명제에 내용을 다루기 위해 변수, 함수를 도입하고 이들 값에 따라 참, 거짓을 판단할 수 있도록 명제 논리를 확장한 논리
- 술어 : 서술어, 대상이나 속성이나 대상 간 관계 기술하는 기호, 참 또는 거짓 값을 갖는 함수

### 술어 논리의 한정사
- 존재 한정사  
- 전칭 한정사

### 술어 논리의 구문
- 함수 : 주어진 인자에 대해서 참, 거짓 값이 아닌 일반적인 값을 반환, 술어나 다른 함수의 인자로 사용
- 항 : 함수의 인자가 될 수 있는 것, 항이 될 수 있는 것 : 개체상수, 변수, 함수 

### 술어 논리의 종류
- 일차 술어논리(FOL) : 변수에만 전칭한정사와 존재한정사를 적용할 수 있는 술어논리
- 고차 술어논리(HOL) : 변수뿐만 아니라 함수나 술어에도 전칭한정사와 존재한정사를 적용할 수 있는 술어 논리

### 술어 논리의 지식표현
- 지식을 표현한 문장에서 대상, 서술어, 한정사 정보 추출
- 기호 정의 및 논리식 표현

### 술어 논리의 추론

#### 술어논리식의 CNF로의 변환 과정
- 1. 전칭한정사와 존재한정사를 맨 앞으로 배치
- 2. 전칭한정사 제거
- 3. 존재한정사가 단독으로 나온다면 대응되는 술어 기호를 참으로 만드는 값을 변수에 대응, 전칭한정사와 결합된 변수와 동시에 나오는 경우에는 존재한정사에 결합된 변수를 해당 술어의 전칭 한정사에 결합된 다른 변수들의 새로운 함수(스콜렘 함수)로 대체

#### 단일화 과정
- 논리 융합을 적용할 때는 대응되는 리터럴이 같아지도록 변수의 값 맞춰주는 과정

# 불확실한 지식의 표현

## 불확실성의 원인
- 1. 약한 인과성과 애매한 인과관계 -> 확신도, 베이즈 정리 사용 표현
- 2. 자연어는 본질적으로 모호하고 부정확 -> 퍼지 이론 사용 표현
- 3. 불완전하거나 결손된 데이터에 기반한 지식 -> 알려지지 않은 것으로 간주하고 근사적 추론
- 4. 모순된 견해와 상충된 지식 -> 지식소스 별로 가중치 부여
- +) 표현대상이 무작위적인 

## 확신도
- 규칙과 사실의 신뢰 정도를 [-1,1] 구간의 값으로 표현 : 1(단정적 신뢰), -1(단정적 불신)

### 확신도 값에 따른 대응 단계
- 1 : 절대 아니다 definitely not
- -0.8 : 거의 확실하게 아니다 almost not
- -0.6 : 아마 아닐 것이다 probably not
- -0.4 : 어쩌면 아닐 것이다 maybe not
- -0.2~0.2 : 모르겠다 unknown
- 0.4 : 어쩌면 그럴 것이다 maybe
- 0.6 : 아마 그럴 것이다 probably
- 0.8 : 거의 확실하게 그럴 것이다 almonst
- 1 : 확실하다 definitely

## 확률기반 불확실성 표현

## 확률
- 어떤 사건이 일어날 가능성

### 확률의 의미
- 1. 상대빈도 확률 = 빈도주의자 확률 : 전체 실험 횟수 대비 관심 사건의 상대적 빈도
- 2. 주관적 확률 : 확신 또는 믿음의 정도

### 결합확률
- A와 B가 동시에 일어날 확률

### 조건부 확률
- B가 주어질 때 A가 일어날 확률 = P(A|B)

### 베이즈 정리
- 사후확률 P(A|B)
- 가능도 P(B|A)
- 사전확률 P(A)
- 증거 P(B)

## 퍼지 이론

### 집합론
- 자연의 단어는 일반적으로 집합을 가리키는 표현
- 일반 집합 : 명확한 경계를 긋고 집합의 원소는 1, 원소가 아니면 0인 소속

### 퍼지이론
- 개념이나 범주가 반드시 이분법적이지는 않다.
- 정도의 문제 -> 퍼지 집합 도입

### 퍼지집합
- 원소가 모임에 어느정도 속한다는 것
- 명제는 참 또는 거짓이 아닌 부분적으로 참, 부분적으로 거짓
- 소속정도는 [0,1] 범위의 실수값으로 표현

### 언어항을 포함한 지식표현
- 소속함수로 표현된 언어항 포함하는 규칙

### 퍼지추론
- 소속함수로 표현된 언어항 사용하는 퍼지규칙들 대상, 기호적 대응을 통한 추론 대신 수치적 추론 가능
- 비퍼지화 : 퍼지 추론의 결과를 실수 값으로 표현하는 것 ex. 무게중심의 값
